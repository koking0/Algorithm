## 乘积最大

给定N个整数A1, A2, ... AN。请你从中选出K个数，使其乘积最大。  

请你求出最大的乘积，由于乘积可能超出整型范围，你只需输出乘积除以1000000009的余数。  

注意，如果X<0， 我们定义X除以1000000009的余数是负(-X)除以1000000009的余数。
即：0-((0-x) % 1000000009)

【输入格式】
第一行包含两个整数N和K。  
以下N行每行一个整数Ai。  

对于40%的数据，1 <= K <= N <= 100  
对于60%的数据，1 <= K <= 1000  
对于100%的数据，1 <= K <= N <= 100000  -100000 <= Ai <= 100000  

【输出格式】
一个整数，表示答案。

【输入样例】
5 3 
-100000   
-10000   
2   
100000  
10000  

【输出样例】
999100009

再例如：
【输入样例】
5 3 
-100000   
-100000   
-2   
-100000  
-100000

【输出样例】
-999999829

CPU消耗  < 1000ms

请严格按要求输出，不要画蛇添足地打印类似：“请您输入...” 的多余内容。

## Ideas

如果 N = K，那么没得选，必须把所有的数字都选上。

我们真正要处理的时候 K < N 的情况。

我们想要的是乘积最大的数，想要乘积最大，那么当然要选择因子比较大的数，所以我们要先对数组A进行一个排序。

假如 K = 2，并且A中有超过两个负数和超过两个正数，那么最大乘积就是max(最小的两个负数相乘，最大的两个正数相乘)。

假如 K = 4，并且A中有超过两个负数和超过两个正数，首先取出 K = 2 时的最大乘积max(最小的两个负数相乘，最大的两个正数相乘)，然后还要再选两个，其实又是 K = 2 的情况。

所以我们可以推广一下，如果 K 是偶数，那么最终的结果其实取决于 A 数组中的负数的个数：
- 如果 A 数组中负数的个数是偶数个，因为我们每次取一对正数或者负数，负数的话负负得正，最终结果一定是正数；
- 如果 A 数组中负数的个数是奇数个，那么当最后取到只剩下一个负数的时候，它肯定要跟一个正数相乘，是一个负数，所以肯定小于正数相乘

最终结果一定是个非负数。

如果 K 是奇数：
- 所有的数字如果都是负数,那么选出来的结果也一定都是负数
- 否则的话，则 A 数组一定至少有 1 个非负数, 那么我们将最大的数取出来, 此时要选的个数就是 k--， k-- 是偶数，那么就又转化为 k-- 是偶数的情况思考
